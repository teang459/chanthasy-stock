## 3. โครงสร้างสิทธิ์ผู้ใช้

### 3.1 4 ระดับสิทธิ์

| ระดับ | ขอบเขต | สิทธิ์หลัก |
|---|---|---|
| **Super Admin** | ทั้งระบบ | สร้าง/ลบสาขา, จัดการสมาชิกทุกสาขา, ดู Audit Log ทั้งหมด |
| **Store Admin** | 1 สาขา (หรือมากกว่า) | จัดการสมาชิกในสาขา, แก้ไขข้อมูลสาขา, อนุมัติทุกอย่างในสาขา |
| **Staff** | ตาม perm flags ที่ admin กำหนด | ขาย, รับเข้า, ปรับสต็อก, ปิดยอด ฯลฯ |
| **Viewer** | อ่านอย่างเดียว | ดูได้ทุกหน้าที่มีสิทธิ์เห็น แต่แก้ไม่ได้ |

### 3.2 Permission Flags (Store Admin กำหนดให้ Staff)

| Flag | ความหมาย |
|---|---|
| `perm_sell` | ขายของ (movement out) |
| `perm_receive` | รับเข้า (movement in) + สร้าง/รับ PO |
| `perm_adjust` | ปรับสต็อก (adjust) |
| `perm_manage_plants` | เพิ่ม/แก้/ลบ ต้นไม้, หมวดหมู่, ซัพพลายเออร์ |
| `perm_view_reports` | ดูหน้า Reports |
| `perm_finance` | บันทึก รายรับ/รายจ่าย ในหน้า Finance |
| `perm_settle` | เปิด/ปิดยอดประจำวัน |

> **เมนูจะซ่อนอัตโนมัติ** ตามสิทธิ์ที่ได้รับ — เห็นเฉพาะที่ใช้ได้

## 5. Workflow ขายของรายวัน

### ขั้นตอนเต็ม (เพื่อให้ Settlement ทำงานได้สมบูรณ์)

```
เช้า ─────────────────────────────────────────────────
  1. เปิดเว็บ → เมนู "ปิดยอด"
  2. กด "เปิดยอดวันนี้" → ใส่เงินสดตั้งต้น → ยืนยัน
       (เงินสดตั้งต้นคือเงินในลิ้นชักก่อนเริ่มขาย)

ระหว่างวัน ──────────────────────────────────────────
  3. ลูกค้าซื้อ → เมนู "รายการสต็อก"
     a. หา plant ที่ขาย (search หรือ scan barcode)
     b. กดปุ่ม "ปรับสต็อก" (icon ลูกศร)
     c. เลือก type = "จ่ายออก"
     d. กรอกจำนวน
     e. เลือกช่องทางชำระ (cash/transfer/credit/other)
     f. (ไม่บังคับ) เลือกลูกค้า — ถ้าเป็นลูกค้าประจำ จะนับยอดสะสมให้
     g. ยืนยัน → stock ลด, movement out บันทึก, ผูกกับ open day อัตโนมัติ

  4. (ไม่บังคับ) ออกใบเสร็จ:
     ไปหน้า "ประวัติเคลื่อนไหว" → หาแถวที่เพิ่งขาย → กด "ใบเสร็จ"
     → กด "พิมพ์ / บันทึก PDF"

ค่ำ ─────────────────────────────────────────────────
  5. เมนู "ปิดยอด"
  6. ระบบโชว์ "เงินสดคาดหวัง" (= ต้น + ขายเงินสด + รายรับ - รายจ่าย)
  7. กด "ปิดยอด" → กรอกเงินสดที่นับได้จริง → ระบบเทียบ
  8. (ไม่บังคับ) กรอกหมายเหตุถ้ามีส่วนต่าง
  9. ยืนยัน → ระบบบันทึก snapshot + lock ทุก movement ของวันนี้
 10. (ไม่บังคับ) กด "พิมพ์ใบสรุป (Z-report)" → PDF
```

### Quick tips
- **Barcode/QR Scanner:** หน้า Stock มีปุ่ม 📷 "สแกน" — ใช้กล้องมือถือสแกน → SKU เติมในช่อง search ทันที
- **ขายลูกค้าทั่วไป:** ไม่ต้องเลือกลูกค้า ปล่อย default "ลูกค้าทั่วไป"

## 6. เปิดยอด / ปิดยอด

### ทำได้ใคร: ผู้ที่มี `perm_settle` (default ของ store_admin คือ true)

### สถานะของวัน
- 🟢 **เปิดอยู่ (open)** — กำลังบันทึกขายระหว่างวัน
- 🔴 **ปิดแล้ว (closed)** — สรุปยอดแล้ว แก้ไข movement ไม่ได้ (super admin override ได้)
- 🟡 **เปิดซ้ำ (reopened)** — Super admin เปิดให้แก้ พร้อม audit note

### สิ่งที่ระบบคำนวณตอนปิดยอด
- ยอดขายรวม = Σ(qty × ราคา) ของ out movements
- ต้นทุน = Σ(qty × cost)
- VAT = ตาม store.vat_rate (inclusive/exclusive ตาม setting)
- กำไรสุทธิ = ขาย − ต้นทุน + รายรับเพิ่ม − รายจ่าย
- เงินสดคาดหวัง = ต้นทุน(เงินสด) + ขายเงินสด + รายรับ − รายจ่าย
- ส่วนต่าง = ที่นับได้จริง − ที่คาดหวัง (≠ 0 = มีปัญหา)

### ใบสรุป Z-report
หน้า ปิดยอด → ประวัติ → กด "ใบสรุป" → พิมพ์/บันทึก PDF

## 7. ใบสั่งซื้อ (PO)

### ทำได้ใคร: ผู้ที่มี `perm_receive`

### Lifecycle
```
draft → submitted → partial → received
                └→ cancelled (จะ block การรับเข้าต่อ)
```

### สร้าง PO ใหม่
1. เมนู "ใบสั่งซื้อ" → กด "สร้าง PO ใหม่"
2. เลือก supplier (ไม่บังคับ), วันที่คาดว่าจะรับ, หมายเหตุ
3. กด "เพิ่มรายการ" → เลือกต้นไม้ → ใส่จำนวน + ต้นทุน (auto-fill จาก plant.cost)
4. ดูยอดรวมที่ท้ายตาราง
5. กด "สร้าง PO" → ระบบสร้าง PO-YYYY-### อัตโนมัติ

### รับของ
1. เมื่อของมาถึง → ไปที่หน้า ใบสั่งซื้อ
2. กด "รายการ" บน PO ที่ต้องการ → จะ expand แสดงรายการ
3. กด "รับเข้า" บนแถวที่ต้องการ → ใส่จำนวน + lot number (ใส่ใน note)
4. ยืนยัน → ระบบ:
   - +stock ของ plant
   - บันทึก movement type='in' พร้อม note "รับเข้าจาก PO-..."
   - ปรับ status PO: partial หรือ received

### Cancel PO
ทำได้ทั้ง draft + submitted + partial — กด "ยกเลิก" → PO เปลี่ยนเป็น cancelled, รับเข้าต่อไม่ได้

## 8. จัดการลูกค้า

### ทำไมต้องเก็บ?
- เห็นยอดสะสม + ครั้งซื้อต่อลูกค้า
- ใบเสร็จมีชื่อ + ที่อยู่ + เลขผู้เสียภาษี (สำหรับ B2B)
- รายงาน Top 10 ลูกค้า

### เพิ่มลูกค้า
เมนู "ลูกค้า" → "เพิ่มลูกค้า" → กรอก ชื่อ + เบอร์ + อีเมล + LINE + ที่อยู่ + tax_id

### ผูกการขายกับลูกค้า
ตอนปรับสต็อก type=out → dropdown "ลูกค้า" — เลือกจาก list (ลูกค้า active เท่านั้น)

### ดูประวัติซื้อ
หน้า ลูกค้า → กด icon นาฬิกาบนแถวลูกค้า → modal แสดงประวัติการซื้อ 200 รายการล่าสุด

## 9. ใบเสร็จ / ใบกำกับภาษี

### ออกใบเสร็จ
หน้า "ประวัติเคลื่อนไหว" → ทุกแถวที่ type='out' มีปุ่ม "ใบเสร็จ" → modal preview → กด "พิมพ์ / บันทึก PDF"

### ใบเสร็จแสดงอะไร
- ชื่อสาขา + ที่อยู่ + เลขผู้เสียภาษี (ถ้ามี)
- ชื่อ + ที่อยู่ + tax_id ของลูกค้า (ถ้าเลือก) หรือ "ลูกค้าทั่วไป"
- เลขใบเสร็จ format `INV-YYYYMMDD-NNNN` (running per store)
- รายการ + จำนวน + ราคา/หน่วย + ยอดรวม
- ถ้า store.vat_rate > 0: title จะเปลี่ยนเป็น **"ใบกำกับภาษี / ใบเสร็จรับเงิน"** และแสดง breakdown มูลค่าก่อน VAT / VAT / รวม

### ตั้ง VAT
ไปที่ Settings → การ์ด "ข้อมูลสาขา" → กรอก:
- เลขผู้เสียภาษี
- อัตรา VAT % (ไทย = 7)
- โหมด: รวม VAT (รีเทล) หรือ แยก (B2B)

## 10. สำหรับ Super Admin

### Admin Panel — หน้าหลัก
แสดงรายการทั้งหมดของ **สาขา (stores)** พร้อม member count, plant count, สถานะ active, VAT mode

### สร้างสาขาใหม่
1. กด "+ เพิ่มสาขา"
2. ระบบ auto-suggest code (STR001, STR002, ...)
3. ใส่ชื่อ + ที่อยู่ + tax_id + VAT + currency
4. ยืนยัน → สาขาใหม่พร้อมใช้งาน (แต่ยังไม่มีสมาชิก)

### เพิ่มสมาชิก
1. ในแถว store → กด "สมาชิก" → expand panel
2. กด "+ เพิ่มสมาชิก"
3. เลือก mode:
   - **เลือกจาก user ที่มีอยู่** → dropdown user ที่ยังไม่เป็นสมาชิกของ store นี้
   - **สร้าง user ใหม่** → email + password + ชื่อ (Edge Function สร้าง auth user + profile ให้)
4. เลือก role: Store Admin / Staff / Viewer
5. (ระบบ preset perm flags ให้อัตโนมัติ — แต่ปรับได้ทุกตัว)
6. ยืนยัน

### แก้ไขสิทธิ์สมาชิก
ในแถวสมาชิก → กด "แก้" → ปรับ role + perm checkboxes → บันทึก

### ลบ user หรือลบ store
- **ลบ store:** ⚠ cascade ลบทุกอย่าง (plants, movements, finance, settlement, members) ของ store นั้น
- **ลบ user:** ลบบัญชี + cascade ลบ memberships (store ตัวเองคงอยู่)

### View as store
กด "ดูร้าน" → ระบบเปลี่ยน currentStoreId → จะเห็นทุกอย่างเหมือนเป็น store_admin ของ store นั้น

## 11. Audit Log

### ใครเห็นอะไร
- **Super Admin:** ทุก event ของทุก store
- **Store Admin:** event ที่ store_id ตรงกับ store ของตัวเอง

### อะไรถูก log
- `store.create / update / delete`
- `member.add / update / remove`
- `profile.role_change` (เลื่อนเป็น super_admin หรือ demote)
- `settlement.reopen` (พร้อม reason)
- `user.create / delete` (จาก Edge Function)

### ใช้ยังไง
เมนู "Audit Log" → filter ตาม store / action type → click "ดูรายละเอียด" → ดู before/after JSON

## 12. Tips ที่ควรรู้

### ติดตั้งเป็นแอป (PWA)
- มือถือ: Safari/Chrome → กด share → "Add to Home Screen"
- คอมพิวเตอร์: Chrome → URL bar มี icon ⊕ → คลิกเพื่อ install
- ใช้งานออฟไลน์ได้บางส่วน (cached assets + last-known data)

### Keyboard shortcuts
- `⌘K` / `Ctrl+K` — focus ช่อง search ใน Topbar

### Bulk import
หน้า Stock → "นำเข้า CSV" → ดาวน์โหลด template → กรอกใน Excel → upload → preview → confirm

### Multi-store
- Super Admin: เห็นทุก store, switcher dropdown ที่ Topbar
- ผู้ใช้ที่เป็นสมาชิกหลาย store: เห็น switcher dropdown เช่นกัน
- เปลี่ยน store → เปลี่ยน symbol สกุลเงิน + ข้อมูลทุกหน้าตาม store ใหม่

### Currency vs Display
- **Store currency** = สกุลเงินจริงของสาขา (เก็บใน DB)
- **Display currency** ใน Settings = ทาง user override ในเครื่องตัวเอง (localStorage, ไม่กระทบ data)

## 13. แก้ปัญหาเบื้องต้น

| อาการ | แก้ |
|---|---|
| ลืมรหัสผ่าน | หน้า Login → "ลืมรหัสผ่าน?" → ใส่อีเมล → ตรวจ inbox |
| Login แล้วเห็นแต่ welcome screen | ยังไม่ได้เป็นสมาชิก store — ติดต่อ admin ให้เพิ่ม |
| ปุ่ม "เพิ่มต้นไม้" หายไป | ไม่มี `perm_manage_plants` — ขอ store admin |
| รายงาน Top Customers ว่าง | ลืมเลือกลูกค้าตอนขาย — ขายครั้งหน้าเลือก dropdown |
| settlement ยอดผิด | ดูใน Movements ว่ามีรายการแปลกๆ หรือ super admin reopen เพื่อแก้ |
